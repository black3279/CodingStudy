 ## 2장데이터 모델과 질의 언어
 - 데이터 모델은 소프트웨어 작성 방법과 해결 문제를 생각해야 하는 방법에도 영향을 미친다
 - 데이터 구조는 JSON XML 문서 RDB 로 저장한다
 - 관계형 모델은 RDB 이며 최근에는 NoSQL 이 탄생하였다
 - NoSQL 은 대규모 데이터 셋이나 매우 높은 쓰기 처리할 시스템에서 쉽게 사용가능하다
 - 무료 오픈 소스가 많으며 특수 질의가 가능하고 풍부한 데이터 표현이 가능하다
 - RDB 의 경우 관계를 맺어 표현해야하는 반면에 NoSQL 은 JSON 표현으로 저장할 수 있다
 - 따라서 어플리케이션과 DB 간의 1 대 1 매칭에는 효율적이라고 할 수 있다
 - 다대다 다대일의 관계와 같이 관계가 발생하는 경우 NoSQL 은 부적합할 수 있다
  ### 네트워크 모델

  ### 문서 데이터베이스
  - 상위 레코드 내에 중첩된 데이터를 저장하며 네트워크 모델과 비슷하다
  - N:1 이나 다대다 관계를 표현할때는 RDBMS 방법을 사용한다
  
## 어떤 데이터 모델이 어플리케이션 코드를 간단하게 할까
- 해당 특성에 따른 데이터 모델을 채택하는 것이 좋다
- 관계를 맺는 방법과 해당 관계 혹은 데이터 특성을 표현하는 방법을 고려하여 정한다
- 해당 데이터베이스에서 지원하는 기능 혹은 오픈 소스나 모듈 기능 등을 고려한다
- 연결성이 많을 경우에는 문서형 보다는 네트워크형(관계형) 모델이 적당하다
- 데이터 지역성을 선언하면 한번에 많은 데이터를 가져올때 이점이 있으나 그 외에는 아니므로 고려하여 사용한다
- RDBMS 도 최근 들어서 JSON 포맷을 지원하기 시작했다
