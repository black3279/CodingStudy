## 멤버 클래스는 되도록 static으로 만들라
- 외부클래스에서 사용할 정도의 클래스라면 TOP 레벨 클래스에서도 사용할 가능성이 높다.
- 비정적 멤버클래스에서 클래스명.this 를 정규화된 this 라고 한다.
- 정적 멤버클래스는 클래스명.클래스명.메소드 와 같은 방법으로 사용한다.
- 위 비정적 멤버클래스의 방법은 클래스명. 하는 순간 외부 클래스도 생성이 되어 메모리 낭비가 생긴다.
- 위와 같은 숨은 참조의 경우, 최악의 경우에는 가비지 컬렉터에 잡히지 않을 수 있다.

## 톱레벨 클래스는 한 파일에 하나만 담으라
- 클래스에 같은 이름의 멤버 클래스가 있으면 컴파일 실패하는 등의 불상사가 생기므로 멤버 클래스를 사용하지 말자.

# 5장 제네릭
## 로 타입을 사용하지 말라
- 클래스나 인터페이스 선언에 타입 매개변수가 쓰이면, 이를 제네릭 클래스 혹은 제네릭 인터페이스 라고 한다.
- List of E 와 같은 식으로 읽는다.
- 제네릭 클래스와 제네릭 인터페이스를 통틀어 제네릭 타입이라고 한다.
- 각각의 제네릭 타입은 일련의 매개변수화 타입을 정의한다.
- 로 타입이란 제네릭 타입에서 타입 매개변수를 전혀 사용하지 않을 때를 말한다.
- List<E> 의 로 타입은 List 이다.

## 로 타입은 안전하지 않다.
- 로 타입을 사용하면 컴파일 에러가 아닌, 런타임 에러가 발생한다.
- 예를 들어 스트링 타입 리스트에 Integer 가 들어갔다가 get 하는 과정에서 런타임 에러가 발생할 수 있다.
- 제인수는 실제 타입 매개변수이다.

## 비한정적 와일드카드 타입
- 와일드카드 (?)  를 쓰면 컴파일 에러가 발생하므로 미리 알 수 있다.

## 비검사 경고를 제거하라
- 제네릭을 사용하기 시작하면 많은 컴파일 에러를 알 수 있다.

## 배열 보다는 리스트를 사용하라
- 배열은 공변이므로 하위 타입은 상위 타입과 함께 변한다.
- 반면에, 리스트의 경우 서로 다른 타입이 있을 때 서로 하위, 상위 관계를 이루지 않는다.
- Object Array 의 경우 컴파일 에러가 나지 않고 런타임 에러가 난다.
