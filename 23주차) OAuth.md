## 20211010 장태진 프로
### OAuth
- third party Application 에 아이디와 비밀번호를 제공하지 않고 인증 방식을 제공하는 것을 목적으로 시작됐따
- 1.0 은 어플리케이션 지원이 부족하여 구현이 복잡했으나 2.0 부터 표준 프로세스가 만들어졌다
- 인증과정은 인증 요청 시 Authorization 코드를 발급하여 Access 토큰을 발급하여 저장한 후 인증을 완료하는 방식이다
- 로그인이 성공 시 Resource Server가 접속한 클라이언트 id 와 Redirect url 을 확인하는데 리소스 서버에서 지닌 부분과 불일치 할 경우 접속을 끊는다
- 리소스 서버는 해당 승인 이후에 특정 scope 까지 접속 가능한 권한을 리턴해준다

### prototype
- 프로토타입 기반 객체지향언어들은 클래스 선언 없이 객체를 생성 가능하다
- 객체를 생성할 때 프로토타입이 결정되는데 결정된 프로토타입 객체는 다른 임의의 객체로 변경할 수 있다
- __proto__ 프로퍼티로 자신의 부모 객체를 가리킨다
- 자신에게 해당 메소드, 프로퍼티 등이 없다면 __proto__ 를 통해 체인을 타고가며 부모 객체를 탐색하게 된다
- 특정 객체의 프로토타입에 새로운 함수 등을 선언해주면 prototype 프로퍼티에 해당 함수가 담기게 되어 공통 함수로도 사용가능하다
